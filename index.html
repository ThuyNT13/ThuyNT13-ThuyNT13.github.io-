Germination

Prior to CSS as we now know it, web designers utilized tables to layout websites. But now, that practice has been slammed as being syntactically incorrect - to the point that people avoid using it even when it's the correct thing to do so, like assembling data into rows and columns. Admittedly, I am committing programming blaspheme by utilizing a table to frame my website, but seeing how we aren't at the CSS part of the lessons, yet, consider this a throwback to the 90's. Bring on the grunge.

I was playing around with the tables assignment for Dev Bootcamp, trying to achieve the Golden Ratio of nesting boxes, and this just presented itself as a way to display information that's easy on the eye and has a natural flow.

Design

There's a semblance of responsive design to it. The table's dimensions conform horizontally to the screen size as the text area is quite fluid. The img above is what keeps it from scrunching further as it has a fixed dimension. But there are definite limitations. It would look better if both the side cells shifted to the bottom when viewed on a mobile. And that's not going to happen with the table tags.

The best reasons for not using tables as layout are upkeep and bandwidth. Making changes in just the CSS is much easier than searching through a complicated HTML page to find all the elements that need to be updated, so saving time for the web developer. The codes created in CSS in external style sheets allows for caching, saving on reloads and time for user to access the page. Also, many of these will most likely be deprecated and phased out in the future for CSS styling.

Progress

Nevertheless, it's a good visual layout upon which to base future projects, at least ones which are text-heavy. There are others to explore, like Ingrid CSS which fluidly responds to various screen sizes, but this was a good test to block out how I want information to flow on a page.

Next up is to create a Contact page which has a form for taking down people's emails (We just learned forms!) and also write an About page, my least favorite part.

Most importantly, utilize CSS to recreate this page structure!
