Evolution

Some changes from the original table-layout idea as it's not so pretty to look at on mobile devices. It is a bit much to squeeze in that information onto a device that only spans your hand.

Progress

Able to figure out some button styles for links. Utilizing border-radius, background, and other little changes, made it stand out. Also, with the a:hover selector, gave buttons a pop when the cursor is over it. Still deciding on whether or not I like the box-shadow effect.

Learned a lot about CSS selectors and combining them. I can use nav > a.btn:hover to target only the buttons for nav elements for the exact hover modifications that I give it. It's a cleaner CSS, using classes sparingly and utilizing the elements that already exist. Took a while to finally discern relationship of descendant, parent, child and sibling elements. But now it make more sense.

More...

The main issue is trying to get my objects to align just where I want it to. Finally fixed social media links to become inline for media queries, I think. Centering is still a big hassle as margin:auto doesn't always work. Resisting float as the display property has so much potential for futzing with. Plus with display, I can play with flexbox!

Getting rid of the ugly default styles for links cause the them to disappear into the text. Present solution feels clutzy. Possibly overkill with button styles.

Need to explore more responsive design techniques. Finding optimal font and image sizes for readibility on mobiles as well as a large computer monitor is one of a few aspects I need to explore. Plus I don't notice any changes on my phone other than font-size.

Not really focusing on font styles, color or other design issues as I'm still troubleshooting layout and CSS. So there's going to be a whole lot of ugly happening and I'm fine with that. Probably will move the Version section to the side and social media icons inline with nav buttons...or wait for Ruby.
