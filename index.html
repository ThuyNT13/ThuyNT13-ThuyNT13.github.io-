Prior to CSS as we now know it, web designers utilized tables to layout websites. But now that practice has been slammed as being syntactically correct - to the point that people avoid using it even when it's the correct thing to do so, like assembling data into rows and columns. So I am admittedly committing programming blaspheme by utilizing a table to frame my website, but seeing how we aren't at the CSS part of the lessons, yet, consider this a throwback to the 90's. Bring on the grunge.

I was playing around with the tables assignment, trying to achieve the Golden Ratio of box nesting, and this just presented itself as a way to display information that's easy on the eye and has a natural flow.

There's an element of responsive design-ness to it as the table's dimensions conform horizontally to the screen size, so all this text in the middle here will scrunch and magically become slimmer. The text area is quite fluid.. The img above is what keeps it from scrunching further as that is a fixed dimension. But there are definite limitations as it would look better if both the side cells moved to the bottom when viewed on a mobile. And that's not going to happen.

At the least, it's a good visual upon which to base future projects, at least ones which are text=heavy. Believe there are other frameworks that are utilized now and and presently exploring grid structures, like the Ingrid CSS which fluidly responds to various screen sizes.
